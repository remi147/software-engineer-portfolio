# Contributing to My Portfolio

Thank you for interest in my portfolio! While this is primarily a personal project, I welcome feedback, suggestions, and discussions.

## 📋 Guidelines

### Code Style
- Follow PEP 8 style guide
- Use type hints for all functions
- Write clear docstrings
- Keep functions focused and small
- Maximum line length: 100 characters

### Testing
- All changes should include tests
- Maintain 90%+ code coverage
- Run `pytest` before submitting feedback
- Include both unit and integration tests

### Documentation
- Update README files for new features
- Document complex logic with comments
- Include examples in docstrings
- Update API documentation if applicable

### Commit Messages
- Use descriptive commit messages
- Reference issues when applicable
- Use conventional commit format
  - `feat:` for new features
  - `fix:` for bug fixes
  - `docs:` for documentation
  - `test:` for tests
  - `refactor:` for refactoring

## 🐛 Bug Reports

If you find an issue:
1. Check existing issues first
2. Provide clear description
3. Include steps to reproduce
4. Share error messages/logs
5. Mention your environment (Python version, OS, etc.)

## 💡 Feature Suggestions

- Describe the feature clearly
- Explain the use case
- Provide examples if applicable
- Discuss potential implementation approach

## 📊 Project Maintenance

### Release Schedule
- New minor versions: Monthly
- Patch updates: As needed
- Major versions: Quarterly review

### Code Review Process
- All code changes reviewed
- Tests must pass
- Documentation must be updated
- No warnings from linters

## 🤝 Code of Conduct

- Be respectful and professional
- Provide constructive feedback
- Give credit for contributions
- Follow open source best practices

## ❓ Questions?

- Open an issue with the `question` label
- Check existing documentation
- Review project README files

---

Thank you for contributing to making this portfolio better!
